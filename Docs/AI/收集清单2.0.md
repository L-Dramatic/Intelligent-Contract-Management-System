## 1️⃣ 核心业务合同（框架协议层）

**📂 存放位置**: `ai-service/knowledge_base/contracts/cmcc_framework/`

> **收集重点**：必须是**“框架协议” (Framework Agreement)**，而非单次采购合同。AI 需要学习“份额管理”和“单价锁定”逻辑。

### 📦 物资集中采购框架协议

* [ ] **中国移动基站/天线集中采购框架协议**
* [ ] **中国移动光缆/光纤采购框架协议**
* [ ] **智能家庭网关/机顶盒采购框架协议**

**🔍 关键条款验证**：

* 
**价格联动机制**：是否包含铜/铅价格波动触发调价的公式？


* 
**份额管理**：是否提及“中标份额”及“不承诺具体采购金额”？


* 
**付款账期**：是否为“到货-初验-终验-质保”结构（如 3-3-3-1 或 80-20）？



### 🏗️ 通信工程施工框架合同

* [ ] **基站建设施工服务框架合同**
* [ ] **传输管线工程施工框架合同**
* [ ] **室分系统集成施工合同**

**🔍 关键条款验证**：

* 
**审计结算**：必须包含“最终结算金额以第三方审计结果为准”条款。


* 
**甲供材管理**：是否包含“甲供物资保管及丢失赔偿”条款？


* 
**隐蔽工程**：是否包含“剥露检查权”及“拍照验收”要求？



### 🛠️ 网络综合代维框架合同

* [ ] **通信基站综合代维服务合同**
* [ ] **铁塔及机房配套维护合同**

**🔍 关键条款验证**：

* 
**计费模式**：是否包含“包年费 + 按次费（工单）”结构？


* 
**SLA 考核**：是否附带《网络维护质量考核办法》（断站时长、发电及时率）？


* 
**人员配置**：是否有“关键岗位不可调换”及“缺员扣款”条款？

## 2️⃣ 创新功能源文件（Input 源）

**📂 存放位置**: `ai-service/knowledge_base/contracts/source_tog/`

> **收集重点**：中国移动作为**乙方**签订的大合同。用于演示“从 ToG 大合同自动生成 ToB 分包招标文件”的创新功能。

* [ ] **某市政府/大数据局智慧城市建设合作协议** (脱敏版)
* [ ] **某银行/国企数据中心(IDC)租赁与服务协议**
* [ ] **某政务云平台建设与运营合同**

**🔍 关键条款验证**：

* 
**背靠背付款**：是否包含“收到业主付款后向合作伙伴付款”的逻辑？


* 
**税率拆分**：是否包含硬件(13%)、软件(6%)、建安(9%)的混合税率表？


* **总包罚则**：包含总工期及总违约金比例（用于 AI 自动计算分包罚则）。

## 3️⃣ 强制性附件（合规红线）

**📂 存放位置**: `ai-service/knowledge_base/contracts/cmcc_attachments/`

> **收集重点**：这是系统的“风控防火墙”。演示时，如果新建合同不勾选这些，AI 必须报错。

* [ ] **廉洁从业承诺书 (中国移动标准版)** 


* *核心点*：商业贿赂导致“永久黑名单”。


* [ ] **通信建设工程安全生产责任书** 


* *核心点*：乙方负全责，甲方不承担连带责任，一票否决。


* [ ] **供应商保密协议 (NDA)** 


* *核心点*：数据不得出境，接触即保密。

## 🚀 导入与处理指令

收集完成后，请许奕执行以下操作：

1. **文件清洗**：
* 确保文件名规范：`类型_地区_年份.pdf` (例: `物资框架_浙江移动_2024.pdf`)
* **剔除图片扫描件**：确保文本可复制（OCR 识别率差会影响演示效果）。


2. **执行导入**：
```powershell
cd ai-service
python scripts/process_documents.py

```


3. **验证测试**：
* 提问 AI：“如果铜价上涨，光缆价格怎么调整？”
* 预期回答：应引用《价格联动机制》中的公式，提及 SMM 均价和 5% 阈值。
