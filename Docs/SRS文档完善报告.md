## 一、新增内容

### 1.1 新增用例：UC-06A 合同变更管理

**位置**：第4.1.6节，插入在UC-06（废弃/终止合同）和UC-07（AI审查）之间

**核心特点**：
- ✅ 只支持甲方（电信运营商）员工发起变更
- ✅ 乙方变更需求通过线下沟通后由甲方在系统中发起
- ✅ 符合1.2.6节"项目范围排除"（不开发乙方门户）

**主要内容**：
1. **业务场景**：
   - 基站租赁合同变更（租金调整、租期延长、站址迁移）
   - 网络建设合同变更（工期调整、金额变更、技术方案调整）
   - 设备采购合同变更（数量调整、规格变更、交付时间调整）

2. **变更类型与审批规则**：
   | 变更类型 | 是否需要审批 | 审批路径 |
   |---------|-------------|----------|
   | 金额增加 | 是（必须） | 按变更后总金额匹配审批流程 |
   | 金额减少 | 是 | 按原审批流程执行 |
   | 时间延长/提前 | 是 | 按原审批流程执行 |
   | 技术方案变更 | 是（需技术评审） | 原审批流程+技术部门会签 |
   | 附件补充 | 否 | 直接归档，无需审批 |
   | 联系人变更 | 否 | 直接更新，记录日志 |

3. **重大变更判定**（需增加法务会签）：
   - 金额增加超过原金额的20%
   - 履约期限延长超过6个月
   - 核心技术指标变更
   - 付款方式重大调整

4. **业务规则**（BR-130至BR-135）：
   - BR-130：合同在特定状态时不能发起变更
   - BR-131：同一合同只能有一个"变更审批中"的申请
   - BR-132：重大变更自动触发法务会签
   - BR-133：自动生成《合同变更协议》
   - BR-134：变更后金额若触发流程升级，必须重新走新流程
   - BR-135：附件补充和联系人变更无需审批

### 1.2 新增数据实体

**位置**：第7.1.1节"核心业务实体"

**新增实体**：
1. **ContractChange（合同变更）**
   - 包含20个字段：变更ID、原合同ID、变更版本号、变更类型等
   - 支持变更前后内容对比（JSON格式存储）
   - 关联审批流程实例

2. **ChangeComparison（变更对比记录）**
   - 详细记录每个字段的变更前后值
   - 自动计算变更幅度

### 1.3 新增典型应用场景

**位置**：第2.4.3节

**场景**：基站租赁合同变更管理
- 业主要求涨租金（2万→2.5万，增幅25%）
- 系统自动判定为重大变更（超过20%）
- 自动增加法务会签节点
- 生成变更协议并通过邮件与乙方线下签署
- 版本管理：v1.0 → v2.0

**业务价值**：
- 变更流程规范化
- 重大变更自动触发法务审查
- 变更历史完整记录可追溯
- 提升变更处理效率50%
