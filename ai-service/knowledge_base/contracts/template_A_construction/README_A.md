# 📂 知识库架构说明书：A类母本 - 通信建设工程合同

**版本**：v1.0 (中国移动浙江公司 2024版)  
**适用场景**：基站土建、机房装修、传输管线铺设、综合布线等工程项目。

---

## 1. 什么是 A 类合同 (Type A Contract)？

**A 类合同**（即**通信建设工程合同**）是电信运营商资本性支出（CAPEX）中占比最大、风险最高、管理最复杂的合同类型。它不同于简单的采购或服务合同，它涉及施工安全、人员管理、隐蔽工程验收以及复杂的资金结算。

### 核心特点 (Key Characteristics)
基于中国移动（CMCC）的业务规范，A 类合同具备以下鲜明的**国企强监管特征**：

1.  **审计定终身 (Audit-Based Settlement)**：
    * 合同签约价仅为“暂估价”。
    * 最终支付金额以**第三方审计机构出具的审计报告**为准，且通常会进行核减。
2.  **安全一票否决 (Safety Veto Power)**：
    * 实行严格的安全生产责任制。一旦发生亡人事故，合同立即终止，且供应商将被列入集团黑名单。
3.  **政治红线：农民工工资专户**：
    * 必须开设专用账户管理农民工工资，严禁拖欠，发包人有权代发。
4.  **人员与品牌锁定**：
    * 中标时的项目经理（PM）和关键设备品牌（甲供/乙供）被严格锁定，擅自更换将面临高额违约金（如更换项目经理罚款 50 万）。

## 2. 目录结构概览

为了优化 AI (RAG) 的检索效率与逻辑隔离，本知识库采用**“法务大脑 + 合规盾牌”**的双层架构设计：

```text
template_A_construction/
├── The_Legal_Core/                  # 🧠 法务大脑：核心法律条款与主权文件
│   ├── CMCC_ZJ_2024_Construction_Master_Contract_Part1.md    # 封面与协议书（定义项目基本面）
│   └── CMCC_ZJ_2024_Construction_Master_Contract_Part2_3.md  # 通用与专用条款（定义权利义务与违约）
│
└── The_Shield/                      # 🛡️ 合规盾牌：强制性附件与执行参数
    ├── CMCC_ZJ_Construction_Compliance_Redlines.md           # 合规红线附件（安全、廉政、工资）
    └── CMCC_ZJ_Construction_Technical_Specs.md               # 技术执行附件（清单、人员、保修期）

```

## 3. 详细内容介绍

### 🧠 文件夹 1：The_Legal_Core (法律核心层)

> **AI 作用**：当用户询问“违约金怎么算？”、“工期延误怎么处理？”时，RAG 引擎优先检索此目录。

#### 📄 `CMCC_ZJ_2024_Construction_Master_Contract_Part1.md`

* **内容**：**合同封面 + 第一部分 合同协议书**
* **功能**：定义合同的“元数据”。
* **关键信息点**：
* **签约主体**：明确发包人（中国移动浙江分公司）与承包人。
* **核心参数**：工程名称、建设地点、**签约合同价**（大写金额）、**工期**（日历天数）。
* **承诺条款**：双方对质量、工期、廉洁的总体承诺。



#### 📄 `CMCC_ZJ_2024_Construction_Master_Contract_Part2_3.md`

* **内容**：**第二部分 通用条款 + 第三部分 专用条款**
* **功能**：定义合同的“执行逻辑”与“违约惩罚”。
* **关键信息点**：
* **专用条款 (Special Conditions)**：这是最核心的部分，包含了所有针对中国移动特性的修改。
* **违约责任**：详细列出更换人员罚款（50万/次）、工期延误罚款（3000元/天）的具体数字。
* **变更估价原则**：定义了工程量变更时如何重新计价。
* **审计条款**：明确“审计核减率 >5% 时，审计费由乙方承担”的霸王条款。



---

### 🛡️ 文件夹 2：The_Shield (合规与执行层)

> **AI 作用**：当进行**合规审查**或**信息填充**时，AI 强制检索此目录，确保不触碰红线。

#### 📄 `CMCC_ZJ_Construction_Compliance_Redlines.md` (合规红线)

* **内容**：**附件12、13、14、15**（原本的补充附件）。
* **功能**：系统的“风控防火墙”。如果不签署这些文件，系统将禁止合同生效。
* **关键附件**：
* **附件12 农民工工资专户**：包含银行账号管理协议。
* **附件14 安全生产协议书**：界定事故责任，明确“乙方全责，甲方免责”。
* **附件15 廉政责任书**：反腐败条款，涉及商业贿赂的永久禁入规则。



#### 📄 `CMCC_ZJ_Construction_Technical_Specs.md` (技术参数)

* **内容**：**附件1-11 + 附件16**（数据表格类附件）。
* **功能**：系统的“数据库”，提供具体的执行细节。
* **关键附件**：
* **附件1 工程项目一览表**：具体的建设规模、面积、层数。
* **附件3 工程质量保修书**：定义具体的保修期限（如：防水工程**5年**，装修工程**2年**）。
* **附件6/7 人员表**：列出项目经理、安全员的姓名与证件号（用于考勤打卡对比）。
* **附件11 暂估价一览表**：列出哪些设备需要二次招标定价。

## 4. AI 检索策略说明 (RAG Strategy)

本布局专门针对 RAG（检索增强生成）进行了优化：

1. **上下文隔离**：将“法律逻辑”（Legal_Core）与“表格数据”（Specs）分离，防止 AI 在回答法律问题时被大量的表格数字干扰。
2. **权重区分**：在进行**风险审查**时，系统会提高 `Compliance_Redlines.md` 的检索权重，确保没有任何一项合规条款被漏判。
3. **精准提取**：当用户要求“生成一份包含防水保修期的条款”时，AI 可以直接定位到 `Technical_Specs.md` 中的附件3，准确提取“5年”这一数字，而不是胡编乱造。
